# async_logger

## 项目定位

`async_logger` 是一个基于 C++17 实现的轻量异步文件日志库。

调用线程负责构造 `LogMessage` 并把消息提交到有界阻塞队列，后台线程负责格式化、写文件、刷新和按大小滚动。

项目重点不是替代成熟日志框架，而是练习：

- 生产者/消费者模型
- 条件变量和有界队列
- 后台线程生命周期
- Block/Drop 过载策略
- flush/stop 语义
- 文件滚动
- CMake 库目标
- CTest 和跨平台 CI

## 配置表

|       配置        |                  含义                  |
| :---------------: | :------------------------------------: |
|    `file_path`    |            基础日志文件路径            |
| `max_queue_size`  |   生产者和 worker 之间的有界队列容量   |
| `roll_size_bytes` | 当前文件超过该软阈值前滚动，0 表示禁用 |
| `overflow_policy` |           队列满时阻塞或丢弃           |
|   `auto_flush`    |       每写一条是否立即刷新文件流       |

## Block 和 Drop 的取舍

- `Block`：队列满时调用线程等待。优点是不主动丢弃；缺点是磁盘变慢时会把背压传到业务线程。
- `Drop`：队列满时立即返回 `false`。优点是业务线程不被日志 IO 长时间阻塞；缺点是高峰期可能丢失日志，调用方应观察 `dropped_count()`。

## 当前限制

- 日志格式固定，不支持自定义 pattern。
- 只有一个后台 worker。
- 文件滚动只按大小，不按日期。
- 不压缩和清理旧日志。
- `flush()` 的推荐前提是没有持续并发提交。
- `log()` 返回 true 只表示成功入队，不保证后来写盘一定成功。
- 后台写失败目前只累计到 `dropped_count()`，没有错误回调。
- 硬终止进程时不能保证 C++ 析构函数运行。
