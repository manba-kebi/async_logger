### 数据通路中的职责

| 模块                 | 唯一主要职责                         |
| -------------------- | ------------------------------------ |
| `LogLevel`           | 表达日志级别并转换为文本             |
| `LogMessage`         | 保存一条日志的结构化数据             |
| `format_log_message` | 把结构化消息格式化为一行字符串       |
| `BlockingQueue<T>`   | 在线程之间安全传递任务               |
| `LogFile`            | 打开、写入、刷新和按大小滚动文件     |
| `AsyncLogger`        | 组织生产者、队列、后台线程和生命周期 |

### 五条核心不变量

写并发代码时，不要只记“这里要加锁”，要明确锁保护的事实。

1. `BlockingQueue` 的 `queue_` 和 `closed_` 只能在 `mutex_` 保护下访问。
2. 每条被计入 `pending_count_` 的消息最终必须且只能减回一次。
3. `pending_count_` 的所有读写都使用 `pending_mutex_`。
4. `LogFile` 的所有跨线程访问都使用 `file_mutex_`。
5. `stop()` 返回时，所有成功入队的消息都已经处理完，后台线程也已经退出。
